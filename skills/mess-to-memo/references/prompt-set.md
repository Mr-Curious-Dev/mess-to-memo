# Prompt Set

## Scope

Identify:

- available source materials;
- time range or discussion range;
- related docs outside Telegram;
- missing or risky inputs;
- execution scope.

## Extraction

Keep:

- backend and API rules;
- business logic;
- mobile behavior;
- UI and UX decisions;
- onboarding information;
- solved issues;
- known problems;
- repeated questions and answers;
- important internal links;
- implementation gotchas.

Ignore:

- casual chat;
- reactions;
- temporary coordination;
- duplicates;
- personal chatter;
- messages without lasting project value.

## Structuring

Turn extracted items into small searchable knowledge units.

## Page Preparation

Build compact project-memory pages from the dataset. When the user wants a handoff package, generate `notion-ready/` with category pages, `processing-report.md`, and `publish-manifest.json`.

## Final Review

Run safety and quality checks before delivery or Notion publication.
